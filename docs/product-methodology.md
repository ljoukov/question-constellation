# Product Methodology

Question Constellation is built around connected question families unlocked by reusable reasoning patterns. The screenshots show the visual direction, but the core educational logic is:

1. The first-time home screen should be subject -> concrete question family/topic -> practice, not thinking move -> questions. Beginners should enter through familiar GCSE content because abstract thinking moves create too much cognitive load before the learner has experienced them.

2. The thinking pattern should feel earned, not selected from a menu. The intended flow is: student tries a real GCSE question, the app repairs or builds the missing reasoning chain, then reveals and names the reusable pattern.

3. Thinking Memory is a post-practice memory and retrieval surface, not the primary first-use entry point. It should contain patterns the learner has discovered or saved after practice.

4. Thinking Memory should be one unified library organized by subject, not isolated subject silos. Subject filters and grouping are useful, but the bigger product idea is transfer: the same reasoning pattern can reappear across different GCSE topics and sometimes subjects.

5. The core methodology is: concrete first -> guided reasoning -> pattern naming -> transfer -> saved memory.

6. The product positioning shifted from "Question Cloud" to Question Constellation because the key idea is not just many questions, but connected question families unlocked by reusable reasoning patterns.

7. The constellation metaphor should imply progressive transfer: nearby questions are close matches; further questions are harder exam-transfer cases using the same hidden logic.

## Product Implications

- Do not make thinking patterns the primary first-use taxonomy.
- Use subjects and GCSE topics as the learner's entry path.
- Reveal pattern names after practice or after guided reasoning.
- Treat Thinking Memory as a growing record of earned patterns.
- Keep transfer visible: nearby examples should feel easier, while distant examples should show how the same hidden logic supports harder exam transfer.
