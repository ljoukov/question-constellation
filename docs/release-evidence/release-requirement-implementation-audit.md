# Independent release-requirement implementation audit

Date: 2026-07-16

This audit checks the durable release worktree against every recovered requirement in
[`docs/rollout-requirements-audit.md`](../rollout-requirements-audit.md). It does not treat an
active model queue, a checked-in plan, or a valid local artifact as evidence that D1/R2 and the
deployed application have been updated.

Status meanings:

- **PASS**: the non-generation implementation or policy is present and independently exercised.
- **PARTIAL**: the contract exists, but required generated data, import, production state, or real
  browser/model acceptance is still outstanding.
- **FAIL**: no conforming implementation exists. There are no known deterministic failures after
  the three bounded fixes recorded below.

## Bounded deterministic fixes made by this audit

1. **Literature card discoverability.** `src/routes/english-literature/+page.svelte` now places a
   signed-in, secondary plot/quotation-card action above the course-section list. It uses the exact
   learner-scoped flashcard session and retains “Choose an essay task” as the main practice action.
2. **Visible 1–3-mark path.** `src/lib/server/subjectLearning.ts` now tracks `quick:` and standard
   `recall:` candidates separately, so both can be visible. The old fallback no longer claims a
   hidden short check is “selected automatically.”
3. **Exact science close-gap contract.** `src/lib/server/personalLearning.ts` now permits close-gap
   creation only for 4–6-mark Biology/Chemistry/Physics questions with a causal chain. The gap UI
   asks short direct causal questions and contains no arrow answer blank or “fill each step” copy.
4. **No unsourced grade prediction.** The former universal raw-percentage-to-grade scale was removed.
   Independent checked work now yields an observed mark rate only after the existing evidence-width
   gate. Compact official June 2024 boundary metadata covers all 17 offerings, but remains context:
   qualification boundaries are not applied to arbitrary question samples.
5. **Content-addressed study-card lineage.** Raw rollout/event JSONL is ignored because it contains
   full session context. Four reconstructed releases already retain commit-safe manifests, and the
   prepared 26 + 13 runner now requires the same manifest for every new artifact: exact passed
   `gpt-5.6-sol`/`max` summaries, disjoint generator/reviewer ids, prompt/model-output/reviewer
   hashes, nonzero event-stream digest/count evidence, and per-card generated -> reviewed ->
   accepted learner-content hashes. Only deterministic catalog/source/offering bindings and choice-key
   normalization are excluded from that learner-content projection. The final gate independently
   requires those manifests plus exactly 1,401 official
   offering/component pairs and 152 selectable deck scopes.
