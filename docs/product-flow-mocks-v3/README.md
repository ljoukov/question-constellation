# High-Fidelity iPad Flow Mocks

These are three independently developed visual directions for the same verified journey. Every
final screenshot is a full operational iPad landscape screen at `1448 × 1086` (`4:3`), not a
marketing page or a collage.

Start with [the comparison and recommendation](COMPARISON.md). The complete route and state logic is
in [the product-flow specification](../product-flow-mocks-v2/RECOMMENDED-FLOW.md).

## What These Screens Resolve

- The signed-in home is a finite trail with one irresistible current question, not a feature grid.
- A real question remains the dominant object from selection through answer entry.
- Checked work leads to improvement and recheck before a transfer question can open.
- The method is represented as attributable links, not an abstract chain browser.
- Progress is qualified by assessed scope and separates later-at-school topics.
- Authentic papers have visible scarcity and reservation state.
- Recall opens from a useful seven-card recommendation with one format choice, not a configuration
  wall.

Earlier boards in `product-flow-mocks-v2` are retained as journey wireframes and audit history. They
are not the visual target.
